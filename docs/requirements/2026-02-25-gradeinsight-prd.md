# GradeInsight（班级成绩分析与可视化平台）PRD（V1 草案）

日期：2026-02-25  
定位：面向任课教师的“成绩数据分析平台”（而非仅成绩单展示），核心在于：**可靠导入 → 可追溯存储 → 灵活筛选 → 可解释的统计与可视化 → 可落地的教学行动（任务清单/补救）**。  
部署形态：公网 VPS，自建数据库，单教师单账号（你已确认 A）。

---

## 1. 目标与非目标

### 1.1 目标（Goals）
1) **多次考试沉淀**：每次导入后，系统长期保存每个学生的历次考试成绩（含题目得分），支持快速回看与对比。  
2) **分析即行动**：支持按“题号 + 得分条件”筛选学生并一键生成“任务清单”，支持提交追踪（点名变色=已交，持久化）。  
3) **强大的数据分析**：提供班级整体与学生个体的趋势分析、分布分析、排名/分位、题目层面的难点识别与异常提示。  
4) **PC + iPad 友好**：表格/列表在 iPad 上可触控、可滚动、关键操作不拥挤。

### 1.2 非目标（Non-goals，先不做）
- 学校级多班多学科多角色权限体系（后续可扩展为多账号，但不作为首版必须）。  
- 自动识别每题满分/题型（除非 Excel 已提供或你手工配置）。  
- 复杂“教务系统对接/自动同步”（先以 Excel 为唯一数据入口）。
- **多班级/多学科切换与管理**（你已确认不需要，后续如要扩展再规划）。

## 2. 用户画像与典型场景

### 2.1 用户画像
- 角色：任课教师（唯一账号），日常需要快速定位薄弱点、生成补救名单、追踪提交，且要能回看学生长期变化。

### 2.2 典型场景（必须覆盖）
1) **导入一次考试**：上传 Excel → 预览解析 → 确认入库 → 自动生成考试概览。  
2) **罚抄/补救**：选择考试 → 输入题号与得分条件（如 `题 51 得分=0`、`得分≤1`、或 `低于本题平均分`）→ 输出名单 → 生成任务清单 → 学生交作业后点名变色 → 支持筛未交。  
3) **学生诊断**：在任何列表中点击学生 → 查看其历次总分趋势、排名趋势、与班均对比、以及各次考试题目层面的强弱变化。  
4) **班级复盘**：查看一次考试的分数分布、分段人数、平均/中位数、最高最低、优秀/及格线情况；查看多次考试趋势。

## 3. 数据来源与导入规范

### 3.1 现有样例已验证
样例文件：`2025-2026学年度第一学期期末考试【八年级】(英语)-八年级7班.xlsx`  
关键表：`得分明细`（含姓名、班级、总分、以及题号列 `1..56`，且存在复合题列如 `51(1)`…`51(5)`，并包含 `51` 汇总列）。

### 3.2 导入能力（P0）
- 支持上传 `.xlsx` 导入为一次考试记录（exam）。  
- 解析后提供“预览确认”：学生数、题目列数、缺失值、非数值异常、重复标识冲突（同一学生 ID 出现多行）。  
- 确认后事务入库：写入失败自动回滚；成功后可追溯来源文件（文件名、hash、导入时间）。

### 3.3 学生唯一标识（必须明确）
系统需要一个跨考试稳定的学生外部 ID。样例中 `准考证号` 与 `自定义考号` **大部分一致，但存在不一致行**，因此建议两列都落库，并在导入预览阶段进行一致性校验与提示：  
- 若两列一致：直接使用该值作为 `external_id`  
- 若两列不一致：**默认以准考证号作为 `external_id`**，并提示你复核；（可选增强）提供手工切换为“以自定义考号为准/沿用历史映射”的能力  
> 若不一致情况较多或存在换号，需要建立“学生映射规则”（P1）。

### 3.4 题目键（Question Key）
- 统一存为字符串：`"1"…"56"`, `"31(1)"`, `"51(3)"` 等。  
- 允许同一题既有汇总列（如 `51`）又有子问列（如 `51(1)`…）并共存。

## 4. 功能范围（P0 / P1 / P2）

### 4.1 P0（MVP 必须有：你核心需求 + 常规关键分析）

#### A) 登录与基础设置
- 单教师单账号登录（用户名/密码）。  
- 支持修改密码；可查看“最近登录/导入”操作记录（最小审计）。

#### B) 学生与班级（最小但可长期使用）
- 班级信息：班级名称（如“八年级7班”）。  
- 学生列表：姓名 + 外部 ID（准考证号/自定义考号）+ 备注（可选）。  
- 去重策略：同班级下外部 ID 唯一；姓名重复允许但需显示外部 ID 辅助识别。

#### C) 考试管理
- 考试列表：名称、日期（可空但建议填）、学科、导入时间、学生数。  
- 考试详情：  
  - 成绩表（可搜索学生）  
  - 基础统计：平均分、中位数、最高/最低、标准差（可选）、优秀/及格人数（阈值可配置）  
  - 分布图：直方图（分段可配，如 10 分一段）
  - （建议 P0）题目均分查询：按题目列出“平均得分/中位数/人数N”，用于定位题目整体情况与学生弱项对比（注意：若未配置满分，不做跨题目‘难度排名’的强结论）
  - （P0）统计口径提示：展示“本次考试不参与统计人数”（如未扫/缺考/总分非数值），避免均分/分布误读

#### D) 学生详情页（“常规但必须”）
点击任一学生进入详情页，至少包含：  
- 历次考试总分折线图（按日期/导入顺序）  
- 历次班内排名趋势（如 Excel 有班次则记录；否则按总分计算名次）  
- 历次考试明细表（每次总分、客观/主观、关键题得分可展开）
- （P0）弱势/良好题目：对选定一次考试，按“题目平均分”判定并输出两列清单（含：题号、个人得分、班级均分、差值）
  - 若该次考试记录为“不参与统计”（如未扫/缺考）：显示醒目状态提示，并默认不输出弱势/良好题目判定（避免基于无效得分误判）

#### E) 题目筛选（你最核心需求）
- 入口：在某次考试详情中进入“题目筛选”。  
- 输入：题号（下拉/搜索，支持 `51` 与 `51(3)`） + 条件（=、≠、<、≤、>、≥、区间、低于本题平均分、高于本题平均分） + 分数值（相对平均分条件不需要输入）。  
- 输出：命中学生名单（姓名 + 外部 ID），支持一键复制/导出（CSV）。  
- 默认仅在“参与统计”的学生范围内筛选；提供开关“包含未扫/缺考”（用于特殊情况排查/点名）

#### F) 任务清单（你最核心需求：点名变色=已交）
- 从题目筛选结果创建任务：自动记录（考试、题号、条件、阈值、创建时间）。  
- 任务详情页显示学生名单：点击姓名切换 `未交/已交`，即时变色并持久化；支持“仅看未交/已交”。  
- 任务统计：总人数、已交、未交；支持导出未交名单。

### 4.2 P1（强分析能力的第一轮增强）
- **多考试对比（班级）**：均分/中位数/及格率趋势图、分布对比（箱线图）。  
- **进步/退步分析**：相对上次考试（或相对均值）的增减分榜单；可按“进步最大/退步最大/稳定”筛选。  
- **题目分析面板**（需题目满分或题目总分口径）：  
  - 每题平均得分/得分率  
  - 得分分布（0 分人数、满分人数、区间）  
  - 题目与总分相关性（识别“区分度弱/反常”题目，提供提示而非绝对结论）  
- **满分配置（可选增强）**：支持为一次考试维护“题目满分/题型/知识点”，用于“做错定义=得分<满分”。  
- **`.xls` 导入**：支持旧版 Excel（或提示用户转换为 `.xlsx`）。  
- **导出报表**：导出考试汇总、学生个人成绩单、题目分析表（Excel/CSV）。

### 4.3 P2（平台化扩展）
- 多教师账号、多班级权限隔离；操作审计完善。  
- 知识点题库、长期画像（知识点掌握度热力图）。  
- 规则引擎：自动生成补救建议（按阈值/连续下降/薄弱题型等）。

## 5. 数据分析指标（建议口径）

> 指标统一遵循“可解释、可行动”。在 UI 上应提供 tooltip 说明，避免误读。

### 5.1 班级层面（每次考试）
- 人数 N  
- 平均分、**中位数**（抗极值）  
- 最高/最低  
- 标准差（衡量离散程度，可选）  
- 分段人数（如 [0,10), [10,20)…）  
- 及格率/优秀率（阈值可配置）

### 5.2 学生层面（跨考试）
- 总分趋势（折线）  
- 排名趋势（折线）  
- 相对班均差值趋势（学生分-班均）

### 5.3 题目层面（每次考试）
在无需满分配置的情况下仍可提供：  
- 平均得分、得分分布（分桶）  
若配置了满分，则可提供：  
- 得分率（平均得分/满分）  
- “失分人数”与“满分人数”

### 5.4 异常提示（建议）
- 某题大面积 0 分/缺失值异常（可能导入问题或题目极难）  
- 某学生某次考试缺失大量题目分（数据质量提示）  
- 题目得分与总分相关性为负（提示“可能反常/需要复核”）

### 5.5 学生弱势/良好题目判定（按题目平均分）

#### 定义（按“某次考试 + 某个题目 question_key”）
- 设该题班级平均分为：`mean_q`  
- 设某学生该题得分为：`score_s_q`  
- 定义差值：`delta = score_s_q - mean_q`

判定规则（默认）：
- `delta < -ε` → 该学生在该题为“弱势题目”
- `delta > +ε` → 该学生在该题为“良好题目”
- `|delta| ≤ ε` → 该学生在该题为“接近平均/一般”

其中 `ε` 为容差（默认 `0.01`，避免浮点/四舍五入造成边界抖动）。

#### 平均分计算口径（已确认）
- 统计样本仅包含“参与统计”的记录：  
  - 学生该题得分必须为有效数值  
  - 若该学生本次考试总分为非数值（如“未扫，不计排名”）：**不纳入班级统计样本**（记录保留，但不参与：班级均分/中位数/分布、题目均分、排名与趋势等统计）

#### 呈现方式（学生详情页）
- 支持选择某次考试（默认最近一次）  
- 输出两张表/两列列表：  
  - “弱势题目”：按 `delta` 从小到大排序（优先显示差距最大的弱项）  
  - “良好题目”：按 `delta` 从大到小排序  
- 每行展示：`question_key`、`score_s_q`、`mean_q`、`delta`

### 5.6 图表与榜单清单（建议：先做 P0，逐步增强）

#### 班级（单次考试）
- 直方图：分数分布（必做，P0）
- 分段柱状：自定义分段人数（P0）
- Top/Bottom 表：最高分/最低分、前 N、后 N（P0）
- （可选）散点图：客观分 vs 主观分（发现“偏科型失分”）（P1）

#### 班级（跨考试）
- 折线图：均分/中位数/及格率随考试变化（P1）
- 箱线图：多次考试分布对比（P1）
- 榜单：进步最大/退步最大（相对上次考试或相对均值）（P1）

#### 学生（跨考试）
- 折线图：总分趋势、排名趋势（必做，P0）
- 条形图：本次相对上次的增减分（P1）
- 对比线：学生分 vs 班均（P0/P1）

#### 题目（单次考试）
- 条形图：每题平均得分（P1）
- （若有满分）条形图：每题得分率 + 失分人数（P1）
- 相关性榜单：题目得分与总分相关性排序（提示“区分度弱/反常”）（P1）

## 6. 交互与界面（PC + iPad）

### 6.1 通用原则
- 触控目标 ≥ 44×44 px；表格支持横向滚动，表头吸顶；关键按钮固定在可触达区域。  
- 列表/表格：支持按姓名/外部 ID 搜索；导出/复制在右上角统一入口。  

### 6.2 核心页面信息架构
- 考试列表 → 考试详情 →（题目筛选 → 创建任务）  
- 任意名单/表格中的学生姓名 → 学生详情（历史趋势）

## 7. 数据模型（逻辑 ER，P0）

实体建议：
- `Class`（班级）
- `Student`（学生，含 external_id）
- `Exam`（考试，含 name/date/subject/source_file_hash）
- `ExamScore`（学生-考试汇总分：total/objective/subjective/rank 等）
- `ExamQuestionScore`（学生-考试-题目：question_key, score_value）
- `Task`（任务：exam_id + question_key + 条件）
- `TaskSubmission`（任务提交：task_id + student_id + status + timestamp）

关键约束：
- `(class_id, external_id)` 唯一（强烈建议）  
- `Exam.source_file_hash` 唯一（避免重复导入）

## 8. 安全、备份与运维（P0）

- HTTPS 反代；仅登录可访问（至少会话 cookie + CSRF 防护）。  
- 限制上传文件类型与大小；导入失败不污染数据库（事务）。  
- 数据库每日备份（`pg_dump`），保留策略可配置（7/30 天）。  
- 操作日志：导入、创建任务、批量标记提交等关键行为可追踪。

## 9. 验收清单（P0）

1) 上传样例 Excel → 预览显示学生数、题目列数 → 确认入库成功。  
2) 打开考试详情：看到成绩表、均分/中位数/分布图。  
3) 点任一学生：能看到历次总分趋势与每次明细，并能在某次考试下看到“弱势/良好题目”（相对题目班级均分）。  
4) 在题目筛选输入 `51` + `= 0`：输出名单正确。  
5) 从筛选结果创建任务：进入任务详情点名变色并持久化（刷新后仍保持）。  
6) 任务页可筛“仅未交”，可导出未交名单。
