# 班级成绩可视化系统（GradeInsight）—需求确认表（草案）

日期：2026-02-25  
目标：在公网 VPS 上部署一个兼容电脑端与 iPad 的班级成绩管理与可视化系统，支持上传 Excel 记录多次考试成绩、趋势分析、题目维度筛选与“罚抄/任务”交付追踪。

## 1) 现有 Excel（已参考本目录样例）

已确认样例文件：  
- `2025-2026学年度第一学期期末考试【八年级】(英语)-八年级7班.xlsx`

工作表（workbook 内）：
- `得分明细`：包含学生基本信息 + 总分 + 客观/主观 + 各题得分（含复合题，如 `31(1)`、`51(3)` 等）
- `班级英语成绩简表`：精简字段（序号/准考证号/姓名/得分/名次等）
- `班级英语成绩汇总`：汇总字段（含班级、学生属性等）

解析建议（用于入库）：以 `得分明细` 为主表；题目列从表头第 13 列开始（`1...56`，含复合题子列），例如表头包含 `51`、`51(1)`…`51(5)`。

## 2) 功能需求确认（✅=已明确 / ❓=需你确认）

| 模块 | 需求点 | 说明/验收口径 | 状态 |
|---|---|---|---|
| 账号与权限 | 登录保护 | 公网部署，**单教师单账号**；未登录不可上传/查看；支持修改密码 | ✅ |
| 班级/学生 | 学生信息沉淀 | 能长期保存每个学生，跨多次考试关联；支持按班级筛选 | ✅ |
| 考试管理 | 上传 Excel 导入一次考试 | 上传→预览解析结果→确认入库；避免重复导入（按文件哈希/考试名称+日期） | ✅ |
| 成绩记录 | 保存“每次考试每个学生成绩” | 至少保存总分；最好保存客观/主观与题目得分明细 | ✅ |
| 班级分析 | 班级整体趋势 | 平均分/中位数/最高最低/及格率（阈值可配）随考试变化 | ✅ |
| 题目分析 | 每题平均分 | 统计每题（含子问）班级平均得分；用于学生“弱势/良好题目”判定 | ✅ |
| 学生分析 | 单个学生趋势 | 点击学生可查看：历次总分趋势、班内排名趋势；并按“题目平均分”标注每次考试弱势/良好题目 | ✅ |
| 题目查询（核心） | 输入题号+得分筛人 | 例如输入“题号=51，得分=0”→筛出名单；支持条件：等于/小于/小于等于/区间 | ✅ |
| 复合题支持 | `51` vs `51(3)` | 既可筛“51 总分”，也可筛“51(3) 子问”；UI 用下拉列出该题可用键 | ✅ |
| 任务/罚抄（核心） | 一键生成名单并追踪提交 | 从筛选结果生成一份“任务清单”；点名字变色=已交；状态写入数据库可追溯时间 | ✅ |
| 任务维度 | 任务与考试/题目绑定 | 同一题可能多次布置（不同日期/不同班级）；需要任务实例化（task_id） | ✅ |
| 导出 | 导出名单 | 支持导出筛选名单/未交名单为 CSV（Excel 导出作为增强） | ✅ |
| 历史回溯 | 可查看历史导入与任务 | 查看某次考试导入详情、某个任务提交情况 | ✅ |

## 3) 关键交互（你描述的场景落地）

### 3.1 “题号 + 得分”筛选
- 输入：题号（如 `51` 或 `51(3)`）+ 条件（默认“等于”；支持低于/高于本题平均分）+ 分数（如 `0`，选择“相对平均分”时可不填）
- 输出：学生姓名列表（可带学号/准考证号），并展示命中人数
- 默认仅筛选“参与统计”的学生；可选开关“包含未扫/缺考”

### 3.2 “罚抄任务”提交标记
- 从筛选结果生成任务：`[考试] [题号] [条件]` 自动命名，可编辑
- 列表中点学生姓名：状态在 `未交/已交` 之间切换；颜色变化；记录操作时间
- 支持筛选：仅看未交 / 仅看已交

## 4) 数据口径（建议默认口径，可按需调整）

- 学生唯一标识：以 Excel 的 `准考证号` 作为 `external_id`（默认口径）。同时两列都入库（`准考证号`、`自定义考号`），导入预览阶段做一致性校验；若不一致：**默认以准考证号为准**，并给出醒目提示以便你人工复核。✅
- 统计参与口径：若某学生本次考试“总分”为非数值（如“未扫，不计排名”），该学生本次考试记录**不参与任何统计**（班级均分/中位数/分布、题目均分、排名与趋势等），但原始记录保留用于追溯。✅
- “做错/弱势”筛选口径（已确认，暂不引入满分配置）：以 `得分阈值/区间` + `低于本题平均分` 为主；后续如确实需要“得分<满分一键筛错”，再增加“每题满分维护”（P1）。✅

## 5) 非功能需求（部署/安全/备份）

| 项目 | 要求 | 建议 |
|---|---|---|
| 兼容性 | 电脑端 + iPad | 响应式布局；触控目标≥44px；表格支持横向滚动 |
| 数据库 | 必须 | PostgreSQL（Docker 容器或宿主机服务） |
| 部署 | 公网 VPS | Docker Compose + 反向代理（Caddy/Nginx）+ HTTPS |
| 备份 | 必须 | 定时 `pg_dump` + 备份到对象存储/另一台机；保留 7/30 天 |
| 上传安全 | 必须 | 登录后上传；限制文件大小；仅允许 xls/xlsx；解析失败可回滚 |

## 6) 需要你先确认的 1 个关键问题（决定架构）

考试日期：**选填（B）**。若未填写日期，则趋势图按“导入时间”排序，并在 UI 上提示“未填写考试日期”。✅
