# Repository Guidelines 贡献指南设计

**日期**: 2026-03-06  
**状态**: 已批准并执行  
**范围**: 根目录 `AGENTS.md`

---

## 目标

将根目录 `AGENTS.md` 改写为面向贡献者的仓库指南，帮助新加入的开发者快速理解项目结构、常用命令、测试方式与协作规范。

## 设计决策

采用“**标准贡献指南 + 仓库协作约束**”方案：

- 保留通用章节：结构、开发命令、代码风格、测试、提交与 PR
- 增补仓库特定要求：**全中文沟通**、**文档沉淀到 `docs/`**、**必要代码注释使用中文**
- 内容控制在简洁范围内，避免写成冗长手册

## 内容来源

- 项目结构来自 `README.md` 与仓库目录
- 开发与部署命令来自 `README.md`、`docs/runbook/deploy.md`
- 测试约定来自 `pytest.ini` 与 `gradebook/tests/`、`worklists/tests/`
- 提交规范来自最近提交历史中的 `feat:`、`fix:`、`docs:`、`style:`

## 成功标准

- 标题为 `Repository Guidelines`
- 使用 Markdown 标题结构
- 内容简洁、可执行、贴合本仓库
- 明确要求协作、文档、注释优先使用中文
